Ukulele Tuesday: Songs Data Analysis App

This project was developed as part of my Master's in Business Analytics at the UCD Michael Smurfit Business School. It was a group project for the Python programming course, where we designed and implemented a data analysis tool tailored to the needs of Ukulele Tuesday, a vibrant community of ukulele enthusiasts in Dublin.

Python
Pandas
Tkinter
Matplotlib


Understanding the Problem

Ukulele Tuesday manages a rich database of songs played during their weekly gatherings. This data spans three CSV files:

  • tabdb.csv: A database with song metadata (e.g., title, year, difficulty).
  • playdb.csv: Records of when songs were played during weekly sessions.
  • requestdb.csv: Logs of who requested specific songs (e.g., audience or group).
The group needed a Python program to:
  1. Query and filter this data easily based on song attributes or play dates.
  2. Visualize trends and patterns in their song collection using clear and interactive charts.
  3. Provide a user-friendly interface while handling errors gracefully.
These functionalities would help them improve their playlists and make data-driven decisions about their performances.

Developing the Solution

As a team, we collaborated to design and build a robust solution, ensuring it met the group’s requirements while showcasing our skills in Python and data analytics.


Core Features

  • Dynamic Querying: The tool allows users to filter songs by attributes like difficulty, year, language, and date range. Results are displayed in customizable tables, making data exploration easy and intuitive.
  • Error Handling: The program checks for missing columns, incorrect data types, and invalid file paths, ensuring a smooth user experience.
  • Interactive Outputs: Results are displayed in tables where users can customize columns and sort the data.

Data Visualizations

To make the data more actionable, the tool generates several plots using matplotlib:
  • Histograms: Distribution of song difficulty levels, Distribution of song durations.
  • Bar Charts: Songs grouped by language, Songs grouped by decade, Songs grouped by tab source (e.g., new, old, or offline).
  • Line Charts: Cumulative trends of songs played over time.
  • Pie Charts: Gender distribution of lead vocalists.

These visualizations allowed Ukulele Tuesday to quickly identify popular trends, analyze diversity in their collection, and make strategic adjustments to their song selection.

Results and Impact

The program delivered significant value to Ukulele Tuesday by transforming raw data into meaningful insights. Some key outcomes include:

  • Better Playlist Management: They can now identify frequently played or rarely requested songs and make more informed decisions about their weekly performances.
  • Deeper Insights: Visualizations revealed patterns such as the dominance of certain song languages and variations in song difficulty levels, helping the group maintain a balanced and engaging collection.
  • Ease of Use: With a clean interface and robust error handling, the tool was intuitive even for users with minimal technical expertise.

Reflections and Future Improvements

This project not only met the group’s requirements but also demonstrated our ability to deliver user-focused solutions. While the tool is fully functional, there are opportunities for future enhancements:

  • Automated Reports: The ability to generate PDF summaries of visualizations for easier sharing.
  • Advanced Filtering: Adding more dynamic filters, such as grouping data by custom categories.
